## Worth attention

- **Sx — open-source package manager for AI skills, MCPs, and commands** — https://github.com/sleuth-io/sx
  A new open-source CLI tool (`sx`) for installing, managing, and distributing AI skills, MCP servers, and custom commands. For anyone building MCP-heavy workflows this directly addresses the "how do I package and share MCPs" gap. Early-stage Show HN, but the concept is immediately relevant to agent tooling work. Worth cloning and evaluating.

- **n8n@1.123.44 — patches 11 critical security issues (vm2, protobufjs, langsmith)** — https://github.com/n8n-io/n8n/releases/tag/n8n%401.123.44
  If you self-host n8n on the 1.x LTS track, upgrade now. vm2 (Node sandbox) critical vulnerabilities can allow sandbox escape; protobufjs issues can allow code injection. 11 critical CVEs total. The 2.x stable track (2.20.11) received only minor bug fixes this cycle.

- **crewAI 1.14.5 — deprecates CrewAgentExecutor, defaults to AgentExecutor** — https://github.com/crewAIInc/crewAI/releases/tag/1.14.5
  Stable release deprecates the old `CrewAgentExecutor` in favor of `AgentExecutor`. If you have CrewAI in any pipeline, audit before next deploy. Also adds Daytona sandbox tooling and `restore_from_state_id` for mid-run state recovery — useful for long-running agent tasks.

- **files.md — zero-dependency open-source Obsidian alternative** — https://github.com/zakirullin/files.md
  A minimal markdown-based note system that works directly on your filesystem with no proprietary sync or plugin ecosystem. Relevant to anyone running a second-brain or markdown knowledge store. No content was pre-loaded in the feed item; direct inspection recommended.
